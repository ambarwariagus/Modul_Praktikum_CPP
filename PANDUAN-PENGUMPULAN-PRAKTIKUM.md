# Panduan Pengumpulan Hasil Praktikum

Dokumen ini berisi panduan untuk pengumpulan hasil praktikum melalui GitHub.
Setiap mahasiswa diwajibkan membuat repository pribadi dan mengunggah seluruh latihan praktikum
beserta bukti hasil eksekusinya.

---

## 1. Pembuatan Repository

Buat repository di GitHub dengan format nama berikut:

`NIM_NAMA_LENGKAP`

Contoh:

`2310112345_Agus_Ambarwari`

Repository harus bersifat PUBLIC agar dapat diperiksa oleh dosen atau asisten praktikum.

---

## 2. Identitas Mahasiswa

Di dalam repository, buat file README.md yang berisi identitas mahasiswa sebagai berikut:

`
Nama        :
NIM         :
Program Studi :
Fakultas    :
`

Contoh:

`
Nama        : Agus Ambarwari
NIM         : 2310112345
Program Studi : Ilmu Komputer
Fakultas    : Fakultas Ilmu Komputer
`

---

## 3. Struktur Repository

Setiap praktikum harus disimpan dalam direktori terpisah.

Contoh struktur repository:

`
NIM_NAMA_LENGKAP
│
├── README.md
│
├── praktikum_1
│   ├── latihan1.cpp
│   ├── latihan1.png
│
├── praktikum_2
│   ├── latihan2.cpp
│   ├── latihan2.png
│
└── praktikum_3
    ├── latihan3.cpp
    ├── latihan3.png
`

---

## 4. Ketentuan Pengumpulan

Untuk setiap praktikum:

1. Kerjakan seluruh latihan dan tugas yang diberikan.

2. Dalam direktori praktikum, simpan:

a. File Source Code

Format penamaan file:

`<nama_latihan>.cpp`

Contoh:

`latihan_vector.cpp`

b. Screenshot Hasil Program

Format penamaan file:

`
<nama_latihan>.png
atau
<nama_latihan>.jpg
`

Screenshot harus menampilkan hasil eksekusi program.

---

## 5. Catatan Penting

- Gunakan penamaan file yang jelas dan konsisten.
- Pastikan program dapat dikompilasi dan dijalankan dengan baik.
- Setiap latihan harus memiliki source code dan screenshot hasil eksekusi.
- Repository harus selalu diperbarui sesuai dengan praktikum yang dikerjakan.
